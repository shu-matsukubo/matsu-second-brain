# 前提

このワークスペースは Obsidian を前提として構成されている。

## 構成

- .codex/agents : エージェント定義
- .codex/tasks : タスク定義
- .codex/skills : スキル定義
- daily : 日々のメモ
- knowledge : 整理済み知識
- feedback : フィードバック

## 行動指針

1. ユーザーの要求を理解する
2. .codex/agents を確認する
3. 適切なエージェントが存在する場合は従う
4. 存在しない場合は .codex/tasks を確認する
5. 適切なタスクが存在する場合は従う
6. 存在しない場合は自身で判断し対応する
7. 定義済みのエージェント・タスク・スキルを優先し、不足する部分のみ自身で補完する。

## ナレッジの考え方

knowledge は Obsidian の Atomic Notes の考え方を前提とする。

- 1つのナレッジは1つの概念を扱う
- 話題が変わる場合は新しいナレッジを作成する
- ファイルを肥大化させず、関連性はリンクで表現する
- ナレッジ同士の接続を増やし、知識のネットワークを育てる

## スキル

skills はタスクの実行を補助するための知識である。
エージェントは必要に応じてスキルを利用する。

## フィードバック

処理中に得られた気付きや改善候補は、必要に応じて feedback/ai-feedback.md に記録する。
記録方法はフィードバック記録スキルを参照する。
