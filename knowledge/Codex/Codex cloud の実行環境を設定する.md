# Codex cloud の実行環境を設定する

Codex cloud の実行環境では、リポジトリの依存関係、開発ツール、環境変数、シークレット、セットアップスクリプトを設定する。リポジトリをチェックアウトした後に、ビルドやテストを再現できる状態を目指す。

## 実行の流れ

クラウドチャットを開始すると、Codex はおおむね次の順で処理する。

1. 選択したブランチまたはコミットのリポジトリをコンテナへチェックアウトする。
2. セットアップスクリプトを実行する。キャッシュ済みコンテナを再開する場合は、任意のメンテナンススクリプトも実行する。
3. インターネットアクセス設定を適用する。
4. エージェントがコード編集と検証を行う。

`AGENTS.md` にプロジェクト固有のビルド、テスト、レビュー手順を記載すると、エージェントは実行時に参照できる。

## 依存関係とセットアップ

一般的なパッケージマネージャーを使うプロジェクトでは、依存関係を自動セットアップできる。追加ツールや複雑な準備が必要な場合は、セットアップスクリプトへ明示する。

セットアップスクリプトとエージェントは別の Bash セッションで動く。セットアップ中の `export` はエージェント実行まで残らないため、継続して必要な値は環境設定または `~/.bashrc` で管理する。

## 環境変数とシークレット

- 環境変数は、セットアップスクリプトとエージェント実行を含むクラウドチャット全体で利用できる。
- シークレットはセットアップスクリプトでのみ利用でき、エージェント実行前に削除される。

エージェント自身が実行時に必要とする値をシークレットへ登録しても、エージェントから直接参照することはできない。値の機密性と利用フェーズを確認して、環境変数とシークレットを使い分ける。

## キャッシュとインターネットアクセス

Codex はコンテナ状態をキャッシュし、再開時には任意のメンテナンススクリプトで依存関係を更新できる。セットアップスクリプト、メンテナンススクリプト、環境変数、シークレットを変更するとキャッシュは自動的に無効化される。

セットアップ中は依存関係を導入するためインターネットを利用できる。エージェント実行中のインターネットアクセスは既定で無効であり、必要に応じて制限付きまたは無制限のアクセスを設定する。
